Autor: David López Ruiz

<<--- Escenario --->>

Nombre: El caso de uso hace referencia a el requisito funcional #16230 de la web Redmine UPM con nombre "Eliminar Cuenta"

Actores: Participan o bien el Cuidador o bien el dueño, depende de quien realize la acción. En ambos casos el procedimiento es el mismo, 
pero por simplicidad se va a suponer que lo realiza el dueño

Tipo: Primario

Precondiciones: El usuario se ha registrado previamente y ha accedido a la gestion del usuario

Pasos:
    1- El usuario solicita a la página web eliminar la cuenta
    2- La página web solicita al usuario que introduzca sus datos para corroborar que es él
    3- La página web vuelve a solicitar confirmación para eliminar la cuenta
    4- El usuario confirma la eliminación de la cuenta
    5- La página web muestra una notificación confirmando la eliminación de la cuenta
    6- La página web cierra sesión y procede con la eliminación de la cuenta

Postcondiciones: La página web elimina la cuenta

Excepciones:
    2.1- Si los datos introducidos son erróneos se vuelve a solicitar que se introduzcan los mismos
        2.1.1- Si los datos introducidos vuelven a ser incorrectos, se cierra la sesión
    3.1- Si se aborta la eliminación se cancela el caso de uso
