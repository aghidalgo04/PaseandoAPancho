<<--- Escenario --->>

Nombre: El caso de uso hace referencia a el requisito funcional #17575 de la web Redmine UPM con nombre "Login con Google"

Actores: Participan o bien el Cuidador o bien el Dueño, depende de quién realize la acción. En ambos casos el procedimiento es el mismo. Por simplicidad, se llamará a cualquier de los roles como usuario.

Tipo: Primario

Precondiciones: El usuario debe de haberse dado de alta posteriormente con Google

Pasos: 
    1- El usuario pide hacer el proceso de login con Google a la página web
    2- La página web solicita al usuario los datos correspondientes
    3- El usuario introduce los datos correspondientes
    4- La página web comprueba si los datos introducidos son correctos
    5- El usuario ha completado el login con Google y es redirigido a su página de inicio

Postcondiciones: El usuario ha inciciado sesión en la página web

Excepciones: 
    2.1- Si el usuario cierra la ventana, el proceso de login con Google se cancela.
    3.1- Si los datos introducidos son erróneos se vuelve a solicitar que se introduzcan los mismos
